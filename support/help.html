常见问题

1、如何使用 XMT_Backend 服务器架构?

首先确定您已经安装了Node.js，然后在您的项目文件夹中，用npm init命令为您的程序创建一个名为 package.json 的文件。 现在，您可以在您的工程目录下加载xmt_backend模块并保存在依赖项列表里。

npm install xmt_backend

模块加载完之后，您就可以在您的文件中引用

import {Application} from "xmt_backend" let app = new Application();

变量 app 就是由 xmt_backend 创建的服务器应用。

2、如何将静态文件加入 XMT_Backend 所构建的服务器应用中?

在您创建了服务器应用之后，调用以下代码，就可以将静态文件加入您的服务器应用中。

app.configStaticServer(directory,options);

directory 是静态的文件的相对路径或绝对路径。 options 是配置选项。 具体使用方法请见 XMT_Backend API 文档

3、如何在 XMT_Backend 中处理不同类型的 HTTP 请求？

在 XMT_Backend 中，提供了对所有的 HTTP 请求的进行处理的API函数。

首先需要在 app中挂载 collection，每一个 collection可以挂载多个 route，每一个 route都可以注册一个对指定的 url的 HTTP请求进行处理的函数。

let app = new Application(); let collection = new app.collection("/user"); let route = new collection("/name"); route.get(function(request,response){ ... })

在以上的程序中，当用户访问 "/user/name" 时，会产生一个 GET 请求，这个请求可以在 route.get() 中被处理，函数中的 request 和 response 参数即为 HTTP 的请求和响应对象。

4、如何在 XMT_Backend 中获取 URL 中的查询参数？

在 route.get(function(request,response){}) 中的 request 对象中包含有 xmt_query 属性。xmt_query 是一个对象，URL 中的查询参数会被解析到 xmt_query 中。

URL为："/?username=jack"

在相应的 route 中会得到 request.xmt_query = { username : jack }

5、如何在 XMT_Backend 中获取 POST 请求中传递的数据？

app 中封装了解析 POST 请求中的JSON数据的函数名为 createBodyParserTask，该函数可将 JSON 数据解析为对象。

request.xmt_body 对象中存放了 POST 请求中的JSON数据。

如果想在 route 中得到 POST 请求传递的数据，只需在 route 所在的 collection 中加入一个 common 中间件。

POST 请求中的数据为 { "username" : "jack" }

collection.common( app.createBodyParserTask() )

此时 request.xmt_body 应为 { username : "jack" }

6、如何在 XMT_Backend 中使用 RESTful URL？

request.xmt_params 是一个对象，其中包含了 RESTful URL 中参数以及对应的值。

// GET /user/tj request.xmt_params.name // => "tj"

7、如何在 XMT_Backend 中使用 MongoDB？

MongoDB 为 Node.js 提供了官方的模块。只需在程序初始化时引用即可。

npm install mongodb import * as mongodb from "mongodb"

关于 mongodb 的详细情况，请参照官方的 API 文档。
